Ruby variables cheatsheet

Apr. 5, 2015

One of our recent projects has been to define the scopes of different variables. The following table will hopefully serve as a quick cheatsheet for anyone struggling with the differences. More detailed descriptions and examples can be found at Tutorialspoint.

Type Format Scope Usage
Global $global Across a program Try to avoid global variables; they can make code seem cryptic, since they're only defined in one place.
Class @@class Across a class Class variables allow for the modification of data across an entire class of objects via one assignment, which can save significant time.
Instance @instance Across an object Instance variables are useful if you want to define information across an object without making it necessarily accessible outside the object.
Local local Across a block Local variables are the most common variable. Their limited scope allows for names to be recycled, which can keep code readable.
Constant CONSTANT Across class, module, or program Constants are meant to remain the same throughout a program and will produce a warning if the program attempts reassignment.
