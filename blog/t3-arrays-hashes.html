Arrays vs. Hashes

Mar. 16, 2015

Arrays and hashes are two basic data structures used in most coding languages, including Ruby. This post is meant to be a cursory comparison of the two types.

Type Declaration (Empty) Declaration (Filled) Accessing chocolate
Array my_array = [] my_array = ["chocolate", "vanilla"] my_array[0]
Hash my_hash = {} my_hash = {best_flavor: "chocolate", worst_flavor: "vanilla"} my_hash["best_flavor"]

An array is indexed to zero, which can be useful for moving though the array, as the index can simply be increased to traverse the data. Arrays can also be sorted, allowing for better organization of data and faster search times.

A hash uses a key-value system, so rather than being indexed to a number, a hash is indexed with a number of specific keys. However, the keys in a has cannot be sorted, so the efficiency we find in writing clearer input-output can be counteracted by inefficiency in organization and sorting.
